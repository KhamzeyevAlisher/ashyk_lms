# Документация модуля: Студенческие лекции (Student Lectures)

Данный модуль отвечает за отображение списка видеоуроков, лекций и учебной статистики в личном кабинете студента. Модуль включает в себя интерфейс фильтрации, блок статистики и динамически генерируемый список карточек лекций.

## 1. Структура файлов

Предполагаемая структура проекта:

*   **`parties/student_dashboard/student_lectures.html`** — Основной HTML-шаблон компонента.
*   **`student_dashboard/lecture.js`**  — Логика рендеринга данных и обработки событий.
*   **CSS** — Стили для карточек, иконок и адаптивности (в данном коде не предоставлены, подразумевается наличие классов `card`, `lecture-card`, `btn-primary` и др.).

---

## 2. HTML Структура

Компонент обернут в контейнер с классом `content-body` и `tab-content`. Внутри `student_lectures.html` выделяются три основные зоны:

### 2.1. Фильтр и Поиск
Верхняя карточка (`.card`) содержит:
*   Заголовок и описание секции (на казахском языке).
*   Поле поиска (`input[type="text"]`) с иконкой лупы.
*   Кастомный выпадающий список (`.custom-select`) с ID `courseSelect` для фильтрации по предметам.

### 2.2. Статистика (Оқу үлгерімі)
Карточка с классом `.purple-card`. Содержит три метрики:
1.  **Барлық дәрістер** (Всего лекций).
2.  **Қаралды** (Просмотрено).
3.  **Аяқталды** (Процент завершения).

### 2.3. Список лекций
Состоит из заголовка списка и контейнера для динамического контента.
*   **Кнопка "Барлық курстар"**: Вызывает функцию `openTab('courses')` (требует реализации внешней функции навигации).
*   **Контейнер рендеринга**: `<div id="lecture-list"></div>`. Именно сюда JavaScript вставляет карточки лекций.

## 3. Структура данных (JavaScript)

Данные хранятся в массиве объектов `lecturesData`.

### Пример объекта данных:

```javascript
{
    category: "Бағдарламалау",          // Категория (отображается над заголовком)
    course: "JavaScript",               // Идентификатор или название курса (для логики перехода)
    title: "JavaScript-тегі ООП...",    // Заголовок лекции
    description: "Объектіге...",        // Краткое описание
    duration: "1 сағ 15 мин",           // Длительность
    date: "22.11.2025",                 // Дата публикации
    files: [                            // Массив прикрепленных файлов (может быть пустым)
        { 
            name: "slides.pdf",         // Имя файла для отображения
            url: "#"                    // Ссылка на скачивание
        }
    ]
}
```

## 4. Логика работы (JS функции)

### `renderLectures(data)`
Основная функция, отвечающая за генерацию HTML-разметки.

1.  **Очистка**: При вызове очищает содержимое контейнера `#lecture-list`.
2.  **Обработка файлов**: Проверяет массив `files`. Если он не пуст, генерирует блок ссылок с иконками скрепок.
3.  **Генерация карточки**: Создает HTML-строку для каждой лекции, подставляя данные из объекта.
4.  **Вставка**: Использует `insertAdjacentHTML('beforeend', ...)` для добавления карточки в DOM.

### Инициализация
Рендеринг запускается автоматически после загрузки DOM:
```javascript
document.addEventListener('DOMContentLoaded', () => {
    renderLectures(lecturesData);
});
```
