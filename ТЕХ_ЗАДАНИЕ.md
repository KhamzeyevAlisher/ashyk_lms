# Техническое задание на разработку системы «OUK LMS»

## 1. Общие положения

### 1.1. Назначение системы
OUK LMS — это программное решение для автоматизации образовательной деятельности вуза, обеспечивающее управление контингентом, планирование учебного процесса и организацию дистанционного взаимодействия между всеми стейкхолдерами (студент – преподаватель – администрация) в рамках единой информационной экосистемы.

### 1.2. Стек технологий и Архитектура
*   **Backend:** Python 3.x, Django 5.x.
*   **Frontend:** HTML5, CSS3 (SCSS), **Vanilla JavaScript (ES6+)**.
    *   Логика интерфейса строится на чистом JS.
*   **Взаимодействие:** SPA-подобная архитектура (Hybrid SPA).
    *   Существует 4 базовых HTML-шаблона (точки входа): `student_dashboard.html`, `teacher_dashboard.html`, `head_teacher_dashboard.html`, `admin_dashboard.html`.
    *   Навигация по табам (вкладкам) внутри роли происходит без перезагрузки страницы через **AJAX (Fetch API)**. Контент подгружается динамически в основной контейнер (`#main-content`).
*   **База данных:** PostgreSQL или mySQL
*   **Языки интерфейса:** Казахский (kk), Русский (ru).

## 2. Детальные функциональные требования

### 2.1. Модуль Аутентификации и Авторизации (Страница входа)
Единая точка входа для всех ролей. Дизайн минималистичный, фокус на форме.

1.  **Форма входа:**
    *   Поля: `Login/Email` и `Password`.
    *   Валидация на клиенте (JS) перед отправкой (проверка на пустые поля).
    *   Валидация на сервере: проверка существования пользователя, проверка хеша пароля.
    *   **Логика перенаправления:** После успешного входа Django определяет роль пользователя (`is_student`, `is_teacher` и т.д.) и редиректит на соответствующую базовую страницу (SPA-контейнер).
2.  **Восстановление доступа:**
    *   Ссылка «Забыли пароль?».
    *   Ввод Email -> отправка ссылки для сброса (Django PasswordResetView).
3.  **Локализация:**
    *   Переключатель языков (KAZ/RUS) в углу экрана. Сохранение выбора в куки/сессии.

### 2.2. Панель Студента (Student Panel)
*URL:* `/student_dashboard/`
*Структура:* Боковая панель (Sidebar) с навигацией. При клике JS перехватывает событие, делает запрос к API, получает HTML-фрагмент и вставляет его в DOM.

*   **2.2.1. Главная (Дашборд / Басты бет)**
    *   **Карточки статистики:**
        *   *Всего тестов:* кол-во доступных тестов.
        *   *Доступные тесты:* активные на данный момент.
        *   *Средний балл (GPA):* расчет на основе журнала. Отображение динамики (например, +2% к прошлому месяцу).
        *   *Прошедшие тесты:* кол-во сданных.
    *   **Виджет «Сегодняшнее расписание»:** список пар на текущую дату (Время, Предмет, Аудитория, Преподаватель).
    *   **Виджет «Актуальные задания»:** список ДЗ с горящим дедлайном.

*   **2.2.2. Лекции (Дәрістер)**
    *   **Поиск и фильтр:** AJAX-поиск по названию предмета или темы. Фильтр по курсам (семестрам).
    *   **Список курсов:** Аккордеон или карточки предметов. При клике раскрывается список тем.
    *   **Просмотр лекции:**
        *   Отображение текста лекции (HTML).
        *   Возможность скачивания прикрепленных файлов (PDF, PPTX).
        *   Кнопка «Изучено» (отправляет AJAX запрос для обновления прогресса).
        *   Прогресс-бар прохождения курса.

*   **2.2.3. Тесты (Тесттер)**
    *   **Список тестов:** Карточки с информацией: Название, Предмет, Дедлайн,  Время на выполнение.
    *   **Статусы:** «Не начат», «В процессе», «Завершен», «Просрочен» (цветовая индикация: серый, синий, зеленый, красный).
    *   **Прохождение теста (Режим Focus Mode):**
        *   Разворачивание на весь экран.
        *   Таймер обратного отсчета (JS).
        *   Пагинация вопросов (по одному или списком).
        *   Типы вопросов: одиночный выбор, множественный выбор.
        *   Автосохранение ответов (отправка ответа при каждом клике или раз в минуту).
        *   Завершение теста: вывод результата (кол-во баллов, процент).

*   **2.2.4. Расписание (Кесте)**
    *   **Виды:** Дневное (список), Недельное (сетка).
    *   **Навигация:** Стрелки «Назад/Вперед» меняют дату/неделю (AJAX подгрузка данных).
    *   **Карточка занятия:** Цветовая дифференциация типов занятий (Лекция - синий, Практика - оранжевый, Лаб - фиолетовый).

*   **2.2.5. Журнал (Журнал)**
    *   **Сводная таблица:** Список предметов текущего семестра.
    *   **Колонки:** Итоговый балл, Процент посещаемости, Оценки за 1/2 рубежный контроль, Экзамен.
    *   **Детализация:** При клике на предмет разворачивается история всех оценок и пропусков по датам.

*   **2.2.6. Домашнее задание (Үй тапсырмалары)**
    *   **Табы:** «Активные», «На проверке», «Оцененные».
    *   **Отправка ДЗ:**
        *   Форма с текстовым полем.
        *   Drag & Drop зона для загрузки файлов.
        *   Валидация размера файла (JS).
    *   **Обратная связь:** Отображение комментария преподавателя и оценки.

*   **2.2.7. Профиль (Профиль)**
    *   Фото студента, ФИО, Группа, ID, Специальность.
    *   Смена пароля.

### 2.3. Панель Учителя (Teacher Panel)
*URL:* `/teacher_dashboard/`
*Особенности:* Акцент на управление контентом и проверку работ.

*   **2.3.1. Главная**
    *   Уведомления о непроверенных ДЗ (счетчик).
    *   Ближайшие занятия (на сегодня).

*   **2.3.2. Управление контентом (Конструктор курсов)**
    *   **CRUD Лекций:**
        *   Форма создания: Выбор предмета, Тема, редактор текста , Загрузка файлов.
        *   Кнопка «Опубликовать / Скрыть».

*   **2.3.3. Тесты (Банк тестов)**
    *   **Создание теста:**
        *   Настройки: Время, кол-во попыток, проходной балл, даты доступа.
        *   Конструктор вопросов: Добавление вариантов ответов, отметка правильного.
    *   **Импорт:** Загрузка Excel/CSV/json/txt/word файла по шаблону для массового добавления вопросов.
    *   **Результаты:** Таблица результатов студентов (Фильтр по группе). Кнопка «Экспорт в Excel/json/txt/word».

*   **2.3.4. Силлабус**
    *   Цифровая версия рабочей программы.
    *   Календарно-тематический план: привязка тем к неделям семестра.

*   **2.3.5. Журнал**
    *   Электронная ведомость. Сетка: Студенты (строки) x Даты (столбцы).
    *   **Функции:**
        *   Выставление оценок (числовые значения).
        *   Отметка посещаемости («н», «+»).
        *   Автоподсчет рейтинга допуска.
        *   Сохранение через AJAX 

*   **2.3.6. Проверка ДЗ**
    *   Список сданных работ.
    *   Модальное окно проверки:
        *   Просмотр файла студента (предпросмотр PDF/IMG).
        *   Поле ввода оценки.
        *   Поле текстового комментария.

*URL:* `/admin_dashboard/` 

*   **2.5.1. Пользователи**

    *   Глобальные константы: даты начала/конца семестров, рубежных контролей, сессий.
    *   Управление праздничными днями (влияет на расписание).

## 3. Модели данных 

Для реализации логики потребуются следующие сущности:

1.  **Users:** Расширенная модель `AbstractUser`. Поля: `role` (student, teacher, dean, admin), `middle_name`, `avatar`, `phone`.
2.  **AcademicStructure:**
    *   `Faculty` (Name, Dean_User)
    *   `Department` (Name, Head_User, Faculty_FK)
    *   `Group` (Name, Course_level, Advisor_User, Department_FK)
3.  **StudyProcess:**
    *   `Subject` (Name, Code, Credits)
    *   `Semester` (Name, StartDate, EndDate)
    *   `AcademicPlan` (Subject, Teacher, Group, Semester) — *связующая таблица*.
4.  **Content:**
    *   `Lesson` (Title, Content, Order, Subject_FK)
    *   `Attachment` (File, Lesson_FK)
5.  **Assessment:**
    *   `Test` (Title, Duration, Subject_FK)
    *   `Question` (Text, Type, Test_FK)
    *   `Answer` (Text, IsCorrect, Question_FK)
    *   `TestResult` (Student, Test, Score, Date)
    *   `Assignment` (Title, Deadline, Subject_FK)
    *   `Submission` (Student, Assignment, File, Grade, Comment)
6.  **Gradebook:**
    *   `Attendance` (Student, Date, Subject, Status, LessonType)
    *   `Grade` (Student, Subject, Value, Type (RK1, RK2, Exam))
7.  **Schedule:**
    *   `TimeSlot` (StartTime, EndTime)
    *   `ScheduleEntry` (Group, Subject, Teacher, Room, DayOfWeek, TimeSlot, Type)

## 4. Требования к Интерфейсу и Дизайну

### 4.1. Стилистика
*   Использовать цветовую схему из скриншотов:
    *   Основной: Темно-синий `#1E293B` (Sidebar).
    *   Акцентный: Ярко-фиолетовый `#6366F1` (Кнопки, активные элементы).
    *   Фон: Светло-серый `#F3F4F6`.
    *   Карточки: Белый `#FFFFFF` с легкой тенью `box-shadow`.
*   Иконки: SVG (например, Heroicons или FontAwesome).

### 4.2. UX/UI Поведение
*   **AJAX Navigation:** Все ссылки в сайдбаре имеют атрибут `data-link`. JS слушатель перехватывает клик, предотвращает дефолтное поведение, делает `fetch(url)`, получает HTML и делает `.innerHTML` в контейнер. Также обновляет `window.history.pushState`.
*   **Loaders:** При ожидании ответа от сервера показывать спиннер или скелетон-лоадер (Skeleton Screen).
*   **Notifications:** Всплывающие уведомления (Toasts) справа сверху при успешном сохранении («Данные обновлены») или ошибке.
