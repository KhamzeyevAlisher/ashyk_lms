# Таблица соответствия требований Moodle

Ниже приведен анализ соответствия функциональности LMS Moodle требованиям, описанным в документе `Памятка.docx`.

| № | Требование из Памятки | Соответствие Moodle | Комментарий / Необходимые действия |
|---|---|---|---|
| **1** | **Наличие серверного оборудования** (24 ядра, 60GB RAM, 1.5TB HDD на 1000 пользователей) | ⚠️ Зависит от инфраструктуры | Moodle сам по себе — это ПО. Для соответствия требованию необходимо развернуть Moodle на серверах с указанной конфигурацией. Moodle хорошо масштабируется. |
| **2** | **Наличие системы хранения данных (СХД)** | ✅ Поддерживается | Moodle поддерживает хранение файлов локально или в объектных хранилищах (S3, и др.), что позволяет использовать внешние СХД. |
| **3** | **Функционирование систем кибербезопасности** | ✅ Поддерживается | Moodle регулярно обновляется для устранения уязвимостей. Требуется настройка WAF, HTTPS и защиты на уровне сервера. |
| **4** | **Наличие систем аутентификации для идентификации личности** | ✅ Полностью поддерживается | Встроенные методы: Логин/Пароль, OAuth2 (Google/MS), LDAP, SAML2. Также возможна интеграция с ЕСУТД или Biometric auth через плагины. |
| **5** | **Наличие системы обнаружения заимствований (Антиплагиат)** | 🔌 Требуется плагин | Moodle имеет стандартный API для плагинов плагиата (Turnitin, Unicheck, StrikePlagiarism и др.). Сама система "из коробки" не проверяет на плагиат. |
| **6** | **Наличие системы онлайн-прокторинга** | 🔌 Требуется плагин | Moodle интегрируется с системами прокторинга (Examus, Olymptrade, ProctorEdu, SEB - Safe Exam Browser). Safe Exam Browser поддерживается "из коробки". |
| **7** | **Платформа управления обучением (LMS), веб-сайт, образовательный портал** | ✅ Полностью соответствует | Moodle является полноценной LMS и образовательным порталом с веб-интерфейсом. |
| **8** | **Автоматизированная система обеспечения кредитной технологии обучения** | ⚠️ Частично / Настройка | Moodle имеет журнал оценок, компетенции, настройки прохождения курса. Для полного соответствия правилам кредитной технологии (GPA, транскрипты) обычно требуется интеграция с АИС (Platonus/Univer) или глубокая настройка. |
| **9** | **Информационная безопасность (ISO 27001, 27002)** | ✅ Поддерживается | Moodle имеет настройки политик паролей, ролей, логирования действий (Log stores). Соответствие стандарту обеспечивается административными мерами и настройкой сервера. |
| **10** | **Структурированный дизайн курса, карта результатов, система оценивания** | ✅ Полностью поддерживается | Moodle позволяет структурировать курсы (темы/недели), настраивать Outcomes (результаты), Rubrics (рубрики) и веса оценок. |
| **11** | **Доступ к учебным материалам, системе тестирования** | ✅ Полностью поддерживается | Основная функция Moodle. Мощный модуль "Тест" (Quiz) с множеством типов вопросов. |
| **12** | **Электронное расписание занятий** | ⚠️ Частично | В Moodle есть Календарь. Но обычно полное расписание импортируется из внешней системы или вносится вручную. Это не специализированный софт для *составления* расписания. |
| **13** | **Система самооценивания** | ✅ Поддерживается | Тесты с обратной связью, Лекции с вопросами, Анкеты. |
| **14** | **Электронная библиотека** | ⚠️ Интеграция | Moodle позволяет загружать файлы и создавать базы данных, но для полноценной библиотеки лучше использовать интеграцию (LTI/Link) с внешними ЭБС. |
| **15** | **Электронный документооборот** | ❌ Нет функционала | Moodle не является СЭД. Можно реализовать через модуль "Заявка" (Dataform/Database), но лучше использовать специализированное ПО. |
| **16** | **Онлайн-регистрация в общежитие** | 🔌 Требуется доработка | Не базовый функционал. Можно реализовать через модуль "Опрос" или "База данных", либо внешний плагин. |
| **17** | **Условия для ДО обучающихся с особыми потребностями** | ✅ Поддерживается | Интерфейс Moodle соответствует стандартам доступности WCAG 2.1 AA. Есть инструменты для слабовидящих (Screen readers compatibility, Atto accessibility checker). |
| **18** | **Система мониторинга качества (анкетирование и т.д.)** | ✅ Поддерживается | Модули "Обратная связь" (Feedback), "Анкета" (Questionnaire), отчеты о деятельности, аналитика обучения. |

## Вывод
**Moodle является отличным выбором** для закрытия большинства образовательных требований (обучение, тестирование, доступ к ресурсам).
Однако для административных функций (расписание, общежитие, документооборот, строгий учет кредитов/GPA) Moodle рекомендуется использовать в связке с **АИС (Деканат)** или реализовать необходимые функции через дополнительные плагины/разработку.
